# Android车载沙盒系统 - 招聘总体方案

## 项目背景

Cook-Hire正在开发车载Android应用沙盒管控系统，基于类似VirtualApp的虚拟化技术，实现三方应用的安全运行和全面管控。

**项目周期**：6个月（5个月开发 + 1个月适配交付）
**工作地点**：杭州余杭区（主要），联调期短期出差上海
**预算范围**：25-35K/月 + 股权激励

## 招聘需求

### 人员配置

| 岗位 | 人数 | 年资 | 薪资 | 侧重点 |
|------|------|------|------|--------|
| Android系统架构师 | 1人 | 10年+ | 30-35K | 系统架构、Framework、Native开发 |
| Android高级工程师 | 1人 | 3-5年 | 25-30K | 应用开发、车机适配、功能实现 |

### 人员分工

**系统架构师**：
- 沙箱系统整体架构设计
- 四大组件虚拟化引擎
- Native Hook和Binder劫持
- 性能优化和稳定性保障
- 技术指导和代码审查

**高级工程师**：
- 应用层功能开发
- 存储隔离和路径映射
- 网络代理和管控策略
- 车机系统适配
- 应用兼容性测试

## 招聘流程

### 时间表（8周）

**Week 1-2：JD发布与简历筛选**
- Day 1-2：JD在各渠道发布
- Day 3-14：持续筛选简历，每天30-50份
- 目标：筛选出30+候选人进入面试

**Week 3-4：初面阶段**
- 安排一面，每天2-3位候选人
- 目标：完成20位候选人一面，筛选出10位进入二面

**Week 5-6：深度面试**
- 安排二面和三面
- 目标：确定4-6位候选人进入Offer阶段

**Week 7-8：Offer发放与入职**
- 薪资谈判、背景调查
- 目标：2位候选人接受Offer并确定入职时间

### 面试流程

```
简历筛选 → 一面 → 二面 → 三面 → HR面 → Offer
```

## 风险与应对

### 风险1：资深架构师难招
**原因**：市场稀缺，大厂竞争激烈

**应对**：
- 提高薪资上限至40K+
- 强调技术自主权和股权激励
- 考虑兼职或顾问形式（项目制）
- 拓展猎头渠道

### 风险2：项目时间紧张
**原因**：招聘周期可能超过8周

**应对**：
- 并行招聘，先招架构师
- 应用层工程师可降低要求，优先潜力型
- 必要时引入外包支持非核心模块

### 风险3：候选人对创业公司顾虑
**原因**：稳定性、福利、发展空间

**应对**：
- 突出项目技术挑战性
- 强调团队大厂背景和股权激励
- 提供试用期灵活安排
- 展示公司愿景和发展前景

### 风险4：地理位置限制
**原因**：杭州余杭区+上海出差

**应对**：
- 强调杭州的生活成本优势
- 上海出差提供补贴
- 考虑远程+定期出差的混合模式
