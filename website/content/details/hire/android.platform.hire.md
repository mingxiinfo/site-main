# 车机端应用管控方案需求整理

## 业务需求
针对车机（Android）实现一个沙盒管控系统，包括支持三方应用的安装、运行时行为监控等管理能力。通过将三方应用安装在针对车机所定制化的沙盒中，做到对三方应用程序完整、全面地管控，从而有效记录、限制三方应用程序的能力，保障车机整体的安全性。整体能力技术方案规划图如下：

![技术方案规划图](/content/details/hire/技术方案规划图.png)

基于上图，我们需要提供**沙箱原生控制层**（来介于系统框架层与系统硬件层之间，控制与原生相关的方法），**沙箱框架层**（为安装在沙箱上的应用提供运行时支持）以及**沙箱应用层**（提供沙箱中所安装应用的文件系统、存储空间等）。

## 能力拆分：
1. 应用虚拟安装 / 启动 / 运行；
2. 多实例运行虚拟化及四大组件虚拟化；
3. 权限与环境代理；
4. 存储隔离与路径映射；
5. Native 组件代理；
6. 网络代理；
7. 管控策略配置及策略解析；

## 人员需求
预计高级 Android 开发工程师两名：
- 完成整体功能开发及测试（5 个月）；
- 完成整体车机产品适配性交付；
- 完成产品按需驻场（上海）联调；
