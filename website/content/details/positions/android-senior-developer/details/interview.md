# Android高级开发工程师 - 面试流程（内部版本）

> 本文档为内部使用，规范面试流程和评估标准。

## 面试总体流程

```
简历筛选 → 一面(技术基础) → 二面(深度挖掘) → 三面(成长潜力) → HR面 → Offer
  1-2天      45分钟           45分钟          30分钟        30分钟    3天
```

## 一面：技术基础（45分钟）

### 面试目标
考察Android应用开发基础能力，确保能独立完成开发任务。

### 面试结构
- 自我介绍和项目经历（10分钟）
- 四大组件和基础知识（15分钟）
- 多线程和网络编程（10分钟）
- 编码和问题解决（10分钟）

### 核心题库

#### 1. 项目经历深挖
**问题**："介绍一下您最有技术深度的项目。"

**关注点**：
- 项目复杂度和技术栈
- 个人承担的职责
- 遇到的技术难题和解决方案
- 项目成果

**追问**：
- "这个项目中最难的技术点是什么？"
- "如果重新做，您会如何改进？"

#### 2. Activity生命周期（必问）
**问题**："Activity A启动Activity B，生命周期回调顺序是什么？"

**标准答案**：
A.onPause → B.onCreate → B.onStart → B.onResume → A.onStop

**追问**：
- "为什么是这个顺序？"（考察对生命周期的理解）
- "如果B是透明的，A的onStop会调用吗？"（不会）
- "onSaveInstanceState在什么时候调用？"

**评分**：
- 5分：完整正确且能解释原因
- 3分：顺序正确
- 1分：顺序错误

#### 3. Service的使用
**问题**："startService和bindService的区别是什么？分别适用于什么场景？"

**评分标准**：
- 5分：说清楚生命周期、通信方式、适用场景
- 3分：知道基本区别
- 1分：不清楚区别

#### 4. Handler机制
**问题**："Handler、Looper、MessageQueue的关系是什么？工作原理是什么？"

**评分标准**：
- 5分：能画出消息机制图，说清楚工作流程
- 3分：知道Handler用于线程通信，了解基本原理
- 1分：只知道如何使用Handler

**追问**：
- "如何避免Handler导致的内存泄漏？"
- "主线程的Looper是何时创建的？"

#### 5. 多线程和协程
**问题**："协程相比线程有什么优势？什么时候用协程？"

**评分标准**：
- 5分：能说出协程的优势（轻量级、结构化并发、可读性）
- 3分：知道协程是什么，用过但理解不深
- 1分：不了解协程

#### 6. 网络请求
**问题**："如何用Retrofit发起一个网络请求？OkHttp的拦截器有什么用？"

**评分标准**：
- 5分：熟悉Retrofit用法，了解拦截器的应用场景
- 3分：用过Retrofit，但不了解原理
- 1分：不熟悉网络框架

#### 7. 数据存储
**问题**："Android有哪几种数据存储方式？分别适用于什么场景？"

**标准答案**：
- SharedPreferences：轻量级键值对
- SQLite/Room：结构化数据
- 文件：大文件、图片等
- DataStore：替代SharedPreferences

**评分**：
- 5分：说出所有方式并能对比优缺点
- 3分：说出主要方式
- 1分：只知道一两种

#### 8. 编码能力（可选）
**问题**："如何实现一个单例模式？有哪几种方式？"

或者

**问题**："给一个字符串数组，找出出现频率最高的字符串。"

**评分**：
- 5分：代码正确，考虑边界情况，有优化意识
- 3分：能写出基本逻辑
- 1分：思路不清楚或代码错误

### 一面评估标准

| 维度 | 权重 | 评分 |
|------|------|------|
| 四大组件 | 30% | |
| 多线程和异步 | 25% | |
| 网络和存储 | 25% | |
| 项目经验 | 20% | |
| **一面总分** | **100%** | |

**通过标准**：总分 ≥ 3.5分 → 进入二面

## 二面：深度挖掘（45分钟）

### 面试目标
考察系统层知识了解程度、学习能力、问题解决能力。

### 面试结构
- Framework基础了解（15分钟）
- 技术深度和学习能力（15分钟）
- 问题分析和解决（15分钟）

### 核心题库

#### 1. Framework基础（探索性提问）
**问题**："您了解Activity的启动流程吗？能大致说说吗？"

**评分标准**：
- 5分：能说出AMS、Zygote、ActivityThread等关键点
- 3分：知道涉及系统服务，但细节不清楚
- 1分：完全不了解

**提示**：如果候选人不了解，不要强求，这是加分项而非必须项

#### 2. Binder通信（探索性）
**问题**："听说过Binder吗？知道它是干什么的吗？"

**评分标准**：
- 5分：知道是IPC机制，了解基本原理
- 3分：听说过，知道用于进程间通信
- 1分：不了解

#### 3. 学习能力考察（重点）
**问题**：
"您是如何学习新技术的？举个例子，比如您是如何学会协程/Jetpack/某个框架的？"

**关注点**：
- 学习方法（看文档、读源码、写Demo、看视频、读书）
- 学习深度（只会用 vs 理解原理）
- 学习主动性（工作需要 vs 自我驱动）

**评分标准**：
- 5分：主动学习，有系统的学习方法，追求原理
- 3分：能学会新技术，但主要靠工作驱动
- 1分：被动学习，不主动探索

#### 4. 技术深度追求
**问题**：
"工作之外您会写代码或研究技术吗？有没有研究过某个技术的底层实现？"

**关注点**：
- 是否有技术热情
- 是否愿意深入学习
- 是否对底层技术感兴趣

**评分标准**：
- 5分：有技术热情，主动深入研究
- 3分：偶尔研究，愿意学习
- 1分：只完成工作，没有额外学习

#### 5. 问题解决能力
**问题**：
"描述一次您遇到的最难调试的Bug，是如何定位和解决的？"

**关注点**：
- 问题分析思路
- 调试方法和工具使用
- 解决问题的能力
- 总结和反思

**评分标准**：
- 5分：思路清晰，方法得当，能独立解决
- 3分：在帮助下能解决，有基本的调试能力
- 1分：解决问题依赖他人

#### 6. 代码质量意识
**问题**：
"您如何保证代码质量？团队有做代码审查吗？"

**关注点**：
- 是否有代码规范意识
- 是否写单元测试
- 是否参与代码审查
- 对代码质量的态度

**评分标准**：
- 5分：有强烈的代码质量意识，主动追求高质量
- 3分：有基本的代码规范，能完成合格的代码
- 1分：对代码质量不重视

### 二面评估标准

| 维度 | 权重 | 评分 |
|------|------|------|
| 系统层理解 | 20% | |
| 学习能力 | 30% | |
| 技术深度追求 | 25% | |
| 问题解决能力 | 15% | |
| 代码质量意识 | 10% | |
| **二面总分** | **100%** | |

**通过标准**：总分 ≥ 3.5分 → 进入三面

## 三面：成长潜力（30分钟）

### 面试目标
评估候选人的软性素质、职业规划、团队契合度。

### 核心问题

#### 1. 职业规划
"您接下来的职业规划是什么？为什么对这个职位感兴趣？"

**关注点**：
- 是否有清晰的目标
- 是否想要技术深度转型
- 是否认可创业公司

#### 2. 学习意愿
"这个职位需要学习Android底层和系统层技术，您是否愿意？"

**关注点**：
- 对底层技术的兴趣
- 学习的意愿和信心
- 是否愿意接受挑战

#### 3. 团队协作
"您更喜欢独立工作还是团队协作？"

**关注点**：
- 沟通能力
- 团队精神
- 是否适合小团队

#### 4. 抗压能力
"如何看待加班和工作压力？"

**关注点**：
- 对工作强度的预期
- 抗压能力
- 是否适合创业节奏

#### 5. 对公司的了解
"您对我们公司和这个项目有什么了解？有什么想问的？"

**关注点**：
- 是否做了功课
- 对项目的兴趣
- 关注的重点

### 三面评估标准

| 维度 | 权重 | 评分 |
|------|------|------|
| 职业规划 | 20% | |
| 学习意愿 | 30% | |
| 沟通协作 | 20% | |
| 抗压能力 | 15% | |
| 创业心态 | 15% | |
| **三面总分** | **100%** | |

**通过标准**：总分 ≥ 3.5分 → 进入HR面

---

## HR面：薪资和背调（30分钟）

### 薪资定级
根据三轮面试综合评分确定级别和薪资：
- A级（≥4.0分）：30-33K + 绩效奖金
- B级（≥3.7分）：27-30K + 绩效奖金
- C级（≥3.5分）：25-27K + 绩效奖金

### 背景调查
- 工作经历核实
- 离职原因
- 学历验证

---

### 综合评分

| 面试轮次 | 权重 | 得分 |
|---------|------|------|
| 一面：技术基础 | 40% | |
| 二面：深度挖掘 | 35% | |
| 三面：成长潜力 | 25% | |
| **综合得分** | **100%** | |

### 录用标准
- **强烈推荐**：综合得分 ≥ 4.0
- **推荐录用**：综合得分 ≥ 3.5 且 无单轮 < 3.0
- **不推荐**：综合得分 < 3.5
